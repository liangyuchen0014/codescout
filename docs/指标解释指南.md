# 指标解释指南

本文用于快速判断 CodeScout RL 训练看板。优先看固定验证集指标；训练 batch 指标只适合诊断趋势和异常。

## generate

- `generate/avg_num_tokens`：每个 rollout 平均生成 token 数。关键诊断指标。持续升高通常意味着回答变长、工具循环变多、成本上升。
- `generate/min_num_tokens` / `generate/max_num_tokens`：本 step 中最短/最长 rollout。辅助看异常短输出和超长输出。
- `generate/std_num_tokens`：生成长度离散程度。高值说明同一 batch 内样本难度或行为差异很大。
- `generate/avg_tokens_non_zero_rewards`：有正 reward 的样本平均长度。用于判断“得分样本是否靠长链路探索获得”。
- `generate/avg_tokens_zero_rewards`：零 reward 样本平均长度。关键诊断指标；高值说明模型花了大量 token 但没有调用正确 finish 或定位错误。

## reward / loss

- `loss/avg_final_rewards`：当前训练 batch 的平均最终 reward。最关键的训练信号，但方差很大；不能单独当论文复现结论。
- `loss/avg_raw_advantages`：未取绝对值的平均 advantage。接近 0 是正常的；长期偏正/偏负需要检查 reward 归一化或 baseline。
- `loss/avg_raw_advantages_abs`：advantage 强度。过低说明组内样本 reward 差异太小，策略更新弱；过高说明更新噪声大。

## policy

- `policy/policy_loss` / `policy/final_loss`：策略优化目标。符号和绝对值不直观，主要看是否长期为 0 或突然爆炸。
- `policy/raw_grad_norm`：梯度范数。关键稳定性指标；尖峰代表该 batch 更新强，持续过高可能不稳定，长期为 0 说明有效学习信号不足。
- `policy/policy_entropy`：输出分布熵。关键行为指标；持续下降说明策略更确定，过低可能过早收敛或探索不足。
- `policy/ppo_clip_ratio`：被 clip 的 token/样本比例。关键 PPO/GSPO 约束指标；长期为 0 表示 clip 基本未生效，可能更新幅度太小或统计口径不敏感。
- `policy/policy_lr`：学习率。冗余但用于确认 scheduler/配置是否生效。
- `policy/policy_update_steps`：每个 batch 的策略更新次数。冗余配置确认项；当前基本恒为 1。

## 最关键指标

1. 固定 validation reward / file-module-entity F1。
2. `loss/avg_final_rewards` 的平滑趋势。
3. `reward/file_reward`、`reward/module_reward`、`reward/entity_reward` 的分层趋势。
4. `generate/avg_tokens_zero_rewards` 与 `generate/avg_num_tokens`。
5. `policy/raw_grad_norm`、`policy/policy_entropy`、`policy/ppo_clip_ratio`。

## 相对冗余指标

- `policy/policy_lr`、`policy/policy_update_steps`：主要确认配置。
- `generate/min_num_tokens`、`generate/max_num_tokens`：单独结论价值有限，配合均值和 std 看。
- `policy/final_loss` 与 `policy/policy_loss`：当前配置下高度相似，保留一个通常足够。
